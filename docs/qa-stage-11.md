## 사용자 발견 이슈 조치

- 음식점 카테고리를 지원 카테고리, 등록 양식, 지도, 챗 검색에서 제거했다.
- 지도 전용 경량 API를 추가해 목록 페이지의 100개 제한과 분리했다.
- 감정 추천 두 번째 문항에 다중 선택 안내 문구를 적용했다.
- 세 번째 여행 스타일은 UI에서 기존 값을 교체하고 API에서도 정확히 한 개만 허용한다.
- 추천 지도 링크는 결과 장소 ID 다섯 개를 전달한다.
- 감정 추천 선택과 결과를 Pinia 세션 상태에 보관해 상세·지도에서 뒤로 갔을 때 복원한다.
- 장소 상세의 `원본 보호` 배지를 제거했다.
- 체크인 만족도 UI, API 필드, ORM 컬럼과 DDL을 제거했다.

## 남은 수동 확인

자동 브라우저 제어 런타임이 초기화되지 않아 실제 클릭 기반 시각 QA는 수행하지 못했다. 아래 항목은 로컬 브라우저에서 마지막으로 확인한다.

1. 360px·768px·데스크톱 폭에서 지도와 감정 결과 카드 겹침 여부
2. 감정 추천 결과 → 상세 → 브라우저 뒤로가기 시 결과 화면 유지
3. 감정 추천 결과 → 지도 → 뒤로가기 시 결과 화면 유지
4. 지도에 수천 개 클러스터가 표시될 때 스크롤·확대 성능
5. 카카오맵 위치 권한 허용·거부 양쪽 흐름
