# 감정 추천 이유 생성

## 처리 원칙

장소 순위는 16차원 감정 벡터의 코사인 유사도로 먼저 확정한다. OpenAI는 순위, 유사도, 장소 ID를 변경하지 않으며 확정된 상위 5개 장소의 설명만 작성한다.

처리 순서는 다음과 같다.

1. Pinecone 또는 SQLite fallback이 상위 5개 장소를 확정한다.
2. 선택 감정, 장소 기본 정보, 유사도, 매칭 키워드 수치만 이유 생성 입력으로 만든다.
3. Responses API를 한 번 호출해 5개 이유를 구조화 응답으로 받는다.
4. 응답의 장소 ID 5개가 원래 결과와 정확히 일치하는지 검증한다.
5. 키 미설정, 시간 초과, API 오류, 스키마 오류, ID 불일치가 있으면 5개 모두 규칙 기반 설명으로 대체한다.

## 모델과 응답

- 기본 모델: `gpt-5-mini`
- API: OpenAI Responses API
- 출력: Pydantic 구조화 출력
- 호출 횟수: 추천 요청당 최대 1회
- 추론 강도: `minimal`
- 출력 토큰 상한: 2,000
- 이유 길이: 장소당 한국어 1~2문장, 최대 180자 스키마 제한
- 요청 제한 시간: 기본 20초

API 응답의 각 항목에는 다음 필드가 추가된다.

```json
{
  "reason": "선택한 감정과 장소 감정 수치를 근거로 만든 추천 이유",
  "reason_source": "openai"
}
```

`reason_source`는 정상 구조화 응답이면 `openai`, 대체 설명이면 `rule`이다. 어느 경우에도 장소 5개의 순서와 유사도는 바뀌지 않는다.

## 환경 변수

```dotenv
OPENAI_API_KEY=
OPENAI_CHAT_MODEL=gpt-5-mini
OPENAI_REASON_TIMEOUT_SECONDS=20
```

키는 백엔드 환경 변수로만 전달하며 프론트엔드 번들에는 포함하지 않는다. 키가 비어 있어도 추천 API는 정상 응답하고 `rule` 설명을 사용한다.

## 개인정보와 근거 제한

OpenAI 입력에는 사용자가 고른 감정 키워드와 상위 5개 장소의 제목, 카테고리, 주소, 유사도, 매칭 수치만 포함한다. IP, 브라우저 지문, 비밀번호, 리뷰 작성자 정보는 전송하지 않는다. 프롬프트에는 입력에 없는 장소 사실을 만들지 않도록 명시한다.
